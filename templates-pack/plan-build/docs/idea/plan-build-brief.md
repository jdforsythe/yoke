# Plan-build brief

Replace this file with your own one-page description of the project. The
planner reads it end-to-end and writes `plan-build-features.json` from it.

## Goal

One paragraph: what you're building, who uses it, and why now.

## Hard constraints

- Stack: e.g. Node 20, Postgres 16, no external services.
- Compatibility: e.g. supports macOS + Linux only.
- Anything you definitely don't want: e.g. no auth in v1.

## Surface area (rough sketch)

A bulleted list of subsystems / files / endpoints you expect the project to
touch. The planner will use this to inventory features.

## Stated acceptance criteria

Anything you already know must be true at the end (e.g. "CLI exits 0 on
`--help`," "API returns 401 on missing token"). The planner will allocate
these to the right feature.

## Open questions for the planner

If you'd like the planner to surface unresolved decisions instead of guessing,
list them here.
