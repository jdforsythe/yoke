# Plan-build-review brief

Replace this file with your own one-page description of the project. The
planner reads it end-to-end and writes `plan-build-review-features.json`.

## Goal

One paragraph: what you're building, who uses it, why now.

## Hard constraints

- Stack, target platforms, anything you definitely don't want.

## Surface area (rough sketch)

A bulleted list of subsystems / files / endpoints the project will touch.

## Stated acceptance criteria

Anything you already know must be true at the end.

## Open questions for the planner

If you'd like the planner to surface unresolved decisions instead of guessing,
list them here.
