# One-shot brief

Replace this file with your own one-paragraph (or short list) description of
what you want built. Keep it focused — one CLI, one script, one document.
Anything bigger belongs in a `plan-build` template.

## Goal

Write a one-paragraph statement of what the artifact does and who uses it.

## Must-haves

- The artifact does X.
- Running `pnpm test` exits 0.
- README is unchanged unless the goal explicitly asks for documentation.

## Non-goals

- Anything else.

## Acceptance test

Describe one concrete way the user can verify the artifact does what was asked
(a CLI command to run, an input/output pair, a screenshot to match).
